Solr

CRM allows for custom fields to be defined at both the Ticket level and the Issue level. Storing additional fields on an ad-hoc basis is a relational database makes searching for records based on those fields very challenging.

To solve this, CRM indexes all Ticket records in Solr for searching. Solr lets us save whatever fields are required for each record and still search across all of the fields. Solr documents do not need to have the same set of fields from one document to the next.

Installing Solr

Solr installation instructions are on the Solr Wiki.

If you are already using Solr for other web applications, or if you are planning on using it for multiple applications, you might look into setting up Multiple Cores in Solr.

CRM ships with a Solr schema ready to drop into your Solr conf directory: crm/scripts/solr/schema.xml.

Configuration

Once Solr is up and running, you can edit your crm/configuration.inc

/**
 * Point to the Solr server
 */
define('SOLR_SERVER_HOSTNAME', 'localhost');
define('SOLR_SERVER_PORT', 8080);
define('SOLR_SERVER_PATH', '/solr/crm');
